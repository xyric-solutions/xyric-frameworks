# Competitive Analysis

Understand a market's competitive landscape, identify opportunities, and inform
strategic decisions. This skill is the **method** — frameworks, matrices, and a
synthesis workflow. The *who-we-are* framing ("our position", "our opportunity",
"how we differentiate") is decided per organization — see [`ADAPT.md`](ADAPT.md) — so the
same method works for any organization, product, or client engagement.

When the our-position specifics are not yet decided, **do not invent them** — produce the
full competitive analysis with the our-position column labeled `US (PLACEHOLDER)` and list
what's missing. Never fabricate our differentiators, pricing, or positioning.

## Our-position inputs

The analysis itself (competitors, features, pricing, positioning) needs **no** organization
specifics — it is filled from sourced research. The our-side specifics only frame the
comparison. Before writing the "Our Opportunity" / "Strategic Recommendations" sections,
you need the organization's differentiators, key messages, promise, target segments, and
business model. These are decided per organization (see [`ADAPT.md`](ADAPT.md)); the intake
questions in [`forms.md`](forms.md) gather them when they aren't already settled. If they're
not available (or this is a pure landscape scan with no "us" yet), use a labeled
`US (PLACEHOLDER)` and list what's missing rather than inventing anything.

## The 5P Framework

Profile each competitor across five dimensions:

| Dimension | Focus |
|-----------|-------|
| **P**ositioning | How they position themselves in the market |
| **P**roduct | Features, quality, technology stack, depth |
| **P**ricing | Price points and model (subscription, one-time, freemium, usage-based) |
| **P**romotion | Marketing channels, content strategy, demand gen |
| **P**eople | Team size, expertise, culture, hiring signals |

## Competitor Categories

| Type | Definition | Priority |
|------|------------|----------|
| **Direct** | Same product, same market | High |
| **Indirect** | Different product, same need | Medium |
| **Potential** | Could enter this market | Low |
| **Substitutes** | Alternative solutions / workarounds | Medium |

## Core Rules

1. **Use multiple sources** — never rely on a single source; cross-reference.
2. **Include indirect competitors** — they may become direct.
3. **Focus on actionable insights** — not just data collection.
4. **No invented data** — every claim is sourced and dated; flag inference as inference.
5. **Update regularly** — quarterly minimum for fast-moving markets.

## When to Activate

**Triggers**:
- "Analyze competitors for [product/service]"
- "Who are our main competitors?"
- "How do we compare to [competitor]?"
- Before launching a new product or service line
- During strategic planning or positioning work

## Workflow

1. **Scope** — define the market, the question, and which categories of competitor matter.
2. **Identify** — build the competitor set (direct, indirect, potential, substitutes);
   surface hidden competitors (see [`reference.md`](reference.md)).
3. **Profile** — run the 5P framework per competitor at the depth in `reference.md`
   (full for top direct, lighter for the rest).
4. **Benchmark** — build the competitive matrix and, where useful, a positioning map.
5. **Synthesize** — SWOT the field, identify the gap, frame *our* opportunity against the
   organization's differentiators.
6. **Recommend** — turn the gap into prioritized, rationalized actions.
7. **Maintain** — keep it a living document; date every refresh.

## Anti-Patterns

| Anti-Pattern | Better Way |
|--------------|------------|
| Relying on a single source | Cross-reference website, reviews, social, reports, job posts |
| Analyzing only direct competitors | Include indirect, potential, and substitutes |
| Ignoring small / new entrants | Watch-list them; they shift fastest |
| Copying a competitor's strategy blindly | Differentiate from your own strengths, don't imitate |
| Analysis paralysis (over-researching) | Timebox; ship an actionable read, then iterate |
| Inventing data to fill the matrix | Mark unknowns as unknown; source everything |
