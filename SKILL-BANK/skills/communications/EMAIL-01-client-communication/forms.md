# Client Communication — Forms & Checklists

Ready-to-use intake, checklists, and the 10 email templates. The sender name, brand voice,
and positioning are decided per organization (see [`ADAPT.md`](ADAPT.md)); this file never
hardcodes them. In the templates, `[Name]` = the recipient and `[Sender]` = the sign-off
name from the organization's decided values.

---

## Communication Intake (the questionnaire for adapting this skill)

Ask as one numbered list and **WAIT** for answers; then record them wherever your team keeps
these decisions. Ask only what's missing.

1. **Sender name** — the name (or initial) that signs these emails (use the signing
   individual where a person signs)?
2. **One-line descriptor** — how the org describes itself in a sign-off / About line?
3. **Brand voice attributes** — 3-5 adjectives the writing should embody (e.g. clear,
   confident, practical)?
4. **Off-limits phrasing** — words or clichés to never use?
5. **Partnership promise** — the core value statement you frame work around?
6. **Differentiators** — what makes this org different (so emails reinforce it)?
7. **Response-time standard** — the org's committed turnaround, if any (else use defaults)?

## Quality Checklist

Before sending any client email:

```
VOICE
[ ] Does it sound like a team member, not a vendor?
[ ] Does it match the organization's brand voice attributes?
[ ] Is it personalized to this client/context?
[ ] Would I send this exact email to a colleague?

CLARITY
[ ] Is the purpose clear in the first line?
[ ] Are next steps obvious (theirs or ours)?
[ ] Can they skim it and get the point?

TONE
[ ] Matched to relationship stage?
[ ] No AI-cringe phrases? Nothing on the organization's avoid list?
[ ] Read aloud — does it sound human?

MECHANICS
[ ] Subject line specific and scannable?
[ ] Sign-off name from the organization's decided sender name?
[ ] Proofread for typos?
[ ] Right people CC'd (and no one extra)?
[ ] Attachments included?
[ ] No invented timelines, costs, or figures — flagged for human input?
```

## Template 1: Progress Update

**When**: Regular status updates (frequency set by client)

```
Subject: [Project Name] — [Key Update/Milestone]

Hey [Name],

Quick update on where we are.

WHAT'S DONE
- [Completed item with specific outcome]
- [Completed item with what it unlocks]

WHAT'S NEXT
- [Next task] — targeting [day/date]
- [Next task]

NEED FROM YOU
[Nothing / Specific item with clear deadline]

[OPTIONAL: HEADS UP — Any context they should know]

Anything I'm missing?

[Sender]
```

## Template 2: Deliverable Submission

**When**: Sending work products for review

```
Subject: [Deliverable Name] Ready — [Project Name]

Hey [Name],

[Deliverable] is ready.

WHAT'S INCLUDED
- [Item and what it does]
- [Item and what it does]

WHAT THIS UNLOCKS
[One sentence on what they can now do / decide / see]

ACCESS: [Link or attachment info]

I'd suggest we walk through this together — [propose specific time] work?
Otherwise, send over any questions and I'll clarify.

[Sender]
```

## Template 3: Meeting Request

**When**: Scheduling discussions

```
Subject: Quick chat — [Topic]

Hey [Name],

Want to align on [topic/purpose]. Should be [X] minutes.

What works for you this week? Some options on my end:
- [Day] [Time] ([their timezone])
- [Day] [Time]

Or grab a time here: [HUMAN INPUT REQUIRED — scheduling/booking link]

[Sender]
```

## Template 4: Issue/Blocker Communication

**When**: Something went wrong or is delayed

```
Subject: [Project Name] — Update on [Issue]

Hey [Name],

Flagging something that came up.

WHAT HAPPENED
[Clear, honest description — no jargon]

IMPACT
[What this means for timeline/deliverable — be specific]

WHAT WE'RE DOING
- [Action 1] — [when]
- [Action 2] — [when]

I'll update you [when] with progress. If you want to talk through this,
I'm available [time/day].

[Sender]
```

## Template 5: Follow-Up (Non-Pushy)

**When**: Checking in on unanswered items

```
Subject: Re: [Original Subject]

Hey [Name],

Circling back on this — no rush if you're swamped.

[One sentence reminder of what you need or where things stand]

Let me know if priorities shifted or if there's anything I can do
to help move this along.

[Sender]
```

## Template 6: Scope Change / Additional Work

**When**: Client asks for something new or scope needs adjustment

```
Subject: [New Request/Scope Item] — Options

Hey [Name],

Got your request on [item]. Here's how I'm thinking about it:

WHAT YOU'RE ASKING FOR
[Restate to confirm understanding]

OPTIONS
1. **Add to current scope** — [impact on timeline/budget]
2. **Replace [existing item]** — [what gets deprioritized]
3. **Phase 2** — [handle after current work completes]

MY RECOMMENDATION
[Option X] because [reasoning tied to their goals].

What feels right to you?

[Sender]
```

## Template 7: Project Closing / Handover

**When**: Wrapping up an engagement

```
Subject: [Project Name] — Wrapped + What's Next

Hey [Name],

We did it. Here's where we landed.

WHAT WE DELIVERED
- [Outcome 1 — in terms of their business impact]
- [Outcome 2]
- [Outcome 3]

WHAT YOU CAN NOW DO
[One sentence on capability they now have]

HANDOVER
- [Documentation location]
- [Access/credentials info if applicable]
- [Support period details: X weeks/months for questions]

It's been great working with your team. [Specific genuine compliment.]

If anything comes up down the road, you know where to find me.

[Sender]
```

## Template 8: Scope Creep Response

**When**: Client asks "can you also..." mid-project

```
Subject: Re: [Their Request]

Hey [Name],

Good idea on [request]. Here's the thing — this wasn't in our original scope,
so let me give you a clear picture:

IF WE ADD THIS NOW
- Timeline: [impact]
- Investment: [HUMAN INPUT REQUIRED — additional cost or "included in retainer"]

IF WE KEEP IT FOR LATER
- Can tackle this in Phase 2 / as a follow-up
- Won't slow down current momentum

I can go either way. What matters more to you right now —
speed to completion or getting this in?

[Sender]
```

## Template 9: Delay Communication

**When**: We're going to miss a deadline or be late

```
Subject: [Project Name] — Timeline Update

Hey [Name],

Being upfront — we're not going to hit [original date] for [deliverable].

WHY
[Honest, brief explanation — technical complexity, dependency, underestimate]

NEW TARGET
[Specific date]

WHAT I'M DOING
[Action you're taking to get back on track]

I should have flagged this earlier. Let me know if this timing creates
problems on your end — I want to make sure we're not blocking anything critical.

[Sender]
```

## Template 10: Value-Add Suggestion

**When**: You see an opportunity to help them further

```
Subject: Idea — [Brief Description]

Hey [Name],

Working on [current project] got me thinking.

WHAT I'M SEEING
[Observation about their business/data/process]

OPPORTUNITY
[What they could gain — be specific]

If this is interesting, I can put together a quick scope. No pressure —
just wanted to flag it while it was fresh.

[Sender]
```

## Success Criteria

- [ ] Client understands purpose immediately
- [ ] Sounds like a team member, not a vendor
- [ ] Matches the organization's brand voice and personalized to client context
- [ ] Action items clear (theirs or ours)
- [ ] No AI-cringe phrases; nothing on the organization's avoid list
- [ ] Sign-off from the organization's decided sender name, not invented
- [ ] Response received in expected timeframe
- [ ] Relationship maintained or strengthened
