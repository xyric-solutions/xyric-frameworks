# Client Communication — Reference

Extended patterns, examples, and guidance. Sender identity, brand voice, and positioning are
decided per organization (see [`ADAPT.md`](ADAPT.md)); the structural patterns below are
universal. In examples, `[Sender]` = the sign-off name from the organization's decided values
(its sender name, or the signing individual).

---

## Tone Adaptation Guide (Detailed)

### By relationship stage

| Stage | Greeting | Sign-off | Formality |
|-------|----------|----------|-----------|
| **Pre-engagement** | "Hi [Name]" | "Best, [Sender]" | Professional, warm |
| **Early project** | "Hey [Name]" | "[Sender]" | Professional, friendly |
| **Deep in project** | "Hey [Name]" or just start | "[Sender]" or "-[Initial]" | Casual, efficient |
| **Long-term partner** | First name or start direct | Initial or nothing | Very casual |

### By situation urgency

| Urgency | Subject line | Tone | Length |
|---------|--------------|------|--------|
| **Urgent** | Start with "URGENT:" | Direct, clear, action-focused | Short |
| **Normal** | Standard format | Balanced | As needed |
| **FYI only** | Include "FYI:" or "(no action needed)" | Light, informative | Brief |

## Template Adaptation Notes

### Progress update adaptations

- More formal client: add a full greeting, "Best regards" sign-off
- Less technical client: simplify language, explain outcomes not tasks
- High-touch client: add more context, offer a call option

### Deliverable submission adaptations

- Self-service client: skip the walkthrough offer, just provide access
- Complex deliverable: add a video walkthrough link
- Milestone deliverable: add context on what's next in the journey

### Meeting request adaptations

- Executive: be more direct, propose a single time
- Uses a scheduling tool: just send the link, skip the options
- Prefers async: offer "or we can do this over email if easier"

### Follow-up adaptations

- Multiple follow-ups: be more direct — "Want to make sure this isn't stuck"
- Decision blocker: add context on impact — "Holding off on X until we hear back"
- Warm relationship: more casual — "Hey, this one still alive?"

## Difficult Situations

### When the client is wrong

Don't be defensive or superior. Be a partner correcting course:

```
"I see where you're coming from. Here's what I'm seeing in the data though —
[evidence]. Might change the approach. Worth a quick call to align?"
```

### When delivering bad news

Lead with it. Don't bury it in paragraph three:

```
"Flagging an issue — [state it clearly]. Here's what happened and what we're
doing about it. [Plan]. Happy to jump on a call if you want to talk through it."
```

### When saying no

Be clear, not wishy-washy:

```
"I don't think we're the right fit for [request] because [honest reason].
What I can do is [alternative] — would that help?"
```

### When following up multiple times

After 2-3 attempts:

```
"Hey [Name] — circling back one more time on this. If priorities have shifted,
totally get it — just let me know and I'll close this out on my end."
```

## AI-Cringe Anti-Patterns (Full List)

| Phrase | Why it's bad | Instead |
|--------|--------------|---------|
| "I hope this email finds you well" | Generic AI opener | Jump straight to the point |
| "Per my last email..." | Passive-aggressive | "Circling back on..." |
| "Please advise" | Corporate robot | "Let me know your thoughts" |
| "Please don't hesitate to reach out" | Robotic, no one hesitates | "Questions? Just reply" |
| "Touching base" | Empty phrase | State the actual purpose |
| "As per our discussion" | Overly formal | "Following our chat..." |
| "I wanted to reach out regarding..." | Windup, no value | Start with the point |
| "At your earliest convenience" | Vague | Specific date or "when you get a chance" |
| "Moving forward..." | Corporate filler | Delete it |
| Lazy sign-off with no warmth | Reads canned | Match the relationship warmth; sign with the decided sender name |

### More anti-patterns

- Starting with weather/weekend small talk (unless the relationship warrants it)
- Apologizing for "taking their time" (you're their partner, not an interruption)
- Multiple exclamation points (one max per email)
- ALL CAPS for emphasis (use bold instead)
- Corporate buzzwords (leverage, synergy, circle back, loop in)
- Hedging language when you're confident ("I think maybe possibly we could...")
- Anything on the organization's avoid list

## Response Time Guidelines

Default standard: respond within 24 hours, even if just to acknowledge. Override with the
org's committed turnaround if the intake specifies one.

| Email type | Target response |
|------------|-----------------|
| Urgent / issue reported | Within 2 hours |
| Question needing an answer | Same business day |
| FYI / update | Acknowledge within 24 hours |
| Decision request | Within 24 hours (even if "need more time") |

**If you can't meet these**: let them know when they'll hear back.
