# Campaign Planner

Plan customer-focused marketing campaigns and multi-week go-to-market (GTM) plans. The
core idea: campaigns are organized around the **audience's problems**, not the
organization's services — every message speaks to the customer's business challenge
first. This skill is the **method**. The specifics it operates on — the audience, the key
messages, the differentiators, the channels, and the offerings — are decided per
organization. See [`ADAPT.md`](ADAPT.md) for what to decide before using this for a given
organization.

When those specifics are not yet decided, **do not invent them** — produce a
clearly-labeled **GENERIC DRAFT** with neutral `PLACEHOLDER` values (e.g. `[PAIN POINT]`,
`[ICP TIER]`, `[CHANNEL]`) and list what's missing. **Never invent** the org's pain
points, audience tiers, messaging, differentiators, channels, founder names, or metrics.

Resolve every concrete value at use time: pain points from the org's differentiators and
key messages, audience from its ICP tiers, tone from its brand voice, channels and pillars
from its content plan, offerings from its service lines. Never hardcode them.

## Campaign Philosophy

> **Core principle**: "We are not here to talk about our services or products — we are
> here to talk about the customer's business."

| Vendor approach | Partner approach |
|-----------------|------------------|
| "We offer N services…" | "You're spending hours every week on a manual task…" |
| "Our solution can…" | "Your team is drowning in [specific problem]…" |
| "Buy our product" | "What if [outcome] were dramatically faster?" |
| Service-catalog marketing | Problem-solution storytelling |

### Positioning principles

1. **Lead with the customer's problem** — never lead with the org's capabilities.
2. **Educate before selling** — build trust through genuine value.
3. **Partner positioning** — an embedded team, not a vendor (frame per the org's business
   model).
4. **Substance over hype** — specific outcomes, not buzzwords.
5. **Methodology as proof** — reference the org's documented approach (its differentiators)
   instead of empty claims.

## Pain-Point Campaign Framework

Campaigns target the **customer's problems**; the org's offerings are the solution,
mentioned only after relevance is established. Build the pain-point set from the org's
inputs — derive each theme from its differentiators and key messages, and map the org's
service lines to each pain point behind the scenes (internal reference only).

| Pain point (the org's) | Campaign theme | Story focus |
|------------------------|----------------|-------------|
| `[Pain point 1]` | `[1-line theme]` | What the customer loses today |
| `[Pain point 2]` | `[1-line theme]` | The cost / risk of the status quo |
| `[Pain point 3]` | `[1-line theme]` | Where the customer can't keep up |
| `[Pain point 4]` | `[1-line theme]` | The foundational gap |
| `[Pain point 5]` | `[1-line theme]` | The competitive / growth opportunity |

> Keep the set to ~3–6 pain points. If the org has fewer differentiators, use fewer
> — do not pad with invented problems.

## GTM Plan Generation Mode

When creating a multi-week GTM plan, **always determine the mode first**.

### Step 1: Detect mode

| Signal | Mode | Meaning |
|--------|------|---------|
| No existing GTM plans | **Fresh Launch** | Brand new — ramp up from zero |
| A plan already covers early weeks and the user asks for "the next N weeks" | **Continuation** | Already live — maintain / scale rhythm |
| User names any week > 1 ("Week 3", "Week 5") | **Continuation** | Already posting — don't reset |
| "Create a new GTM plan" with no context | **Ask** | "Is this a fresh launch or a continuation of your current plan?" |

### Step 2: Apply the right cadence

**Fresh Launch** (Week 1 = first-ever week of posting):

| Week | Cadence (per account) | Purpose |
|------|-----------------------|---------|
| Week 1 | Lighter (~5 posts) | Launch the voice, establish presence, test formats |
| Week 2 | Ramp (~8–10 posts) | Build rhythm, expand formats, add channels |
| Week 3+ | Full (~10–12 posts) | Full cadence — all channels firing |

**Continuation** (already posting — Week 3+):

| Week of plan | Cadence (per account) | Purpose |
|--------------|-----------------------|---------|
| Every week | Full (~10–12 posts) | Full cadence from day 1 — no ramp |
| Scale option | Higher (~12–15 posts) | If team bandwidth allows, push higher |

> Treat the numeric cadence above as a default heuristic. If the org or user defines a
> different posting volume, use theirs.

### Step 3: Continuation context check

Before generating a continuation plan, **always check**:

1. **Previous GTM plans** — read existing GTM plans wherever the org stores them (resolve
   the location from where the org keeps its GTM plans; for per-founder plans, one per
   founder).
2. **Themes already used** — don't repeat weekly themes from previous plans.
3. **Formats already used** — track cumulative format usage to maintain diversity.
4. **What performed** — if KPIs have "Actual" data, lean into what worked.
5. **"What Comes Next" section** — previous plans may state explicit Week 3+ guidance —
   honor it.

### Step 4: Continuation plan rules

| Rule | Detail |
|------|--------|
| No ramp-up | Week 1 of the plan = full cadence |
| Theme continuity | New themes only — reference previous themes as "already covered" |
| Cumulative format tracking | Show total format usage across ALL plans, not just current |
| Long-form / podcast cadence | Already established — maintain weekly or bi-weekly |
| Cross-account sync | Same weekly theme, one angle per account / founder — continue the pattern |
| Scaling | Introduce new content types (carousels, newsletters, longer threads, collabs) |
| Reference previous KPIs | Set targets relative to previous weeks, not from zero |

### GTM plan output structure

Every generated plan (fresh or continuation) includes, per week:

1. **Weekly theme sync** (one angle per active account / founder)
2. **Socials table** (Day, Channel, Format, Content, Owner)
3. **Format diversity check** (zero repeats within the same week)
4. **Campaigns** (active, new, evolving)
5. **Partnerships** (research → outreach → active)
6. **PR & long-form** (pipeline status)
7. **Audio / podcast pipeline** (recording → episodes → clips), if applicable
8. **Weekly KPIs** (targets based on previous actuals when available)
9. **Week objectives** (5–7 specific goals)

> **Anti-pattern**: Never generate a continuation plan with Week 1 lighter cadence. If the
> org has been posting for 2+ weeks, the next plan starts at full speed.

## Campaign Types

### Type 1: Thought Leadership
- Duration: ongoing (quarterly themes)
- Primary: top of the org's active channels, all active from Week 1
- Content flow: Insight → Problem → Methodology → Story → Engagement
- Cadence: 5–10 posts/week (company + founders), 1–2 long-form/month

### Type 2: Problem-Focused Lead Generation
- Duration: 4–6 weeks per campaign
- Funnel: ATTRACT (educational) → EDUCATE (methodology) → CONNECT (discovery call)
- Assets: long-form content, a post series (5–7 posts), a booking page

### Type 3: Customer-Story Amplification
- Duration: 2–3 weeks per story
- Approach: focus on the PROBLEM and TRANSFORMATION, not the org
- Use anonymized patterns, before/after stories, lessons learned

## Channel Strategy

Drive the channel list from the org's active channels; the table below is a generic shape.
Map each channel to a purpose and cadence.

| Channel (the org's) | Purpose | Cadence (default) |
|---------------------|---------|-------------------|
| Primary professional network | Thought leadership, brand building, lead gen | 5–10 posts/week (company + founders) |
| Short-form video | Brand awareness, culture, trending | From Week 1 — 1–2/week, scale to 3–5/week |
| Real-time micro-posts | Real-time takes, community | From Week 1 — 1–4/week, scale to daily |
| Owned content / SEO | Long-form educational content | 2–4 articles/month |
| Audio / podcasts | Authority building, new audiences | 2–3 appearances/month |

**Content mix (60/30/10)**: 60% Core (pillar-aligned with hooks/stories), 30%
Personality (BTS, hot takes, human moments), 10% Wildcard (off-topic, trending, polls).
Pull pillars from the org's content pillars.

**Format diversity rule**: Never post the same format twice in a row. Maintain a format
library and rotate (a content-diversity skill, if present, validates this).

## Audience Targeting for Campaigns

Every campaign targets a specific tier from the org's ICP tiers. Lead each tier with the
matching message from the org's key messages — do not invent tier names or messaging.

| Tier (the org's ICP tiers) | Messaging lead |
|----------------------------|----------------|
| `[Tier name]` | `[Lead message from the org's key messages]` |

## Differentiators in Campaigns

When a campaign needs a credibility proof point (e.g. a speed, quality, or outcome
claim), pull it from the org's differentiators — never fabricate metrics, timelines,
or guarantees. If the org has no proof point for a claim, mark it `PLACEHOLDER` and
flag it for confirmation.

## Messaging Principles

| Sound like | Don't sound like |
|------------|------------------|
| "Here's what we've seen work…" | "Our cutting-edge solutions…" |
| "This problem costs more than you think" | "We offer comprehensive services" |
| "Let me show you the approach" | "Contact us to learn more" |
| "We've helped teams like yours" | "We're the leading provider" |

Adapt the exact phrasing to the org's brand voice attributes and avoid anything on its
brand voice "avoid" list.

## Anti-Patterns

| Don't | Instead |
|-------|---------|
| Lead with services | Lead with the customer's problem |
| Use hype language | Use specific outcomes |
| Create service catalogs | Organize by the customer's problems |
| Skip problem quantification | "You're losing [X] because of [Y]" |
| Ignore the org's methodology | Reference the org's differentiators |
| Chase engagement bait | Create genuine value |
