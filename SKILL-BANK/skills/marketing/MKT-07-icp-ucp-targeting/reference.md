# ICP & UCP Targeting — Reference

Extended reference for building tier profiles, writing tier-specific messaging, and proving
UCP pillars. The audience and positioning specifics are decided per organization (see
[`ADAPT.md`](ADAPT.md)); the frames below are universal and you fill them with your own values.

---

## The positioning inputs

These are the values to decide per organization (gather them via the Positioning Intake in
[`forms.md`](forms.md)):

- **promise** — one-line brand promise / core value statement.
- **key messages** — 5–9 canonical messages, each with its best context.
- **ICP tiers** — list of `{ name, description }` — NO pricing.
- **UCP pillars** — list of `{ name, meaning, proof }` — competitive positioning.
- **differentiators** — fallback pillar set if you have no formal UCP pillars.

Resolve every tier, pillar, message, and the promise from your own decided values at use time.
If something isn't decided yet, treat it as unanswered (ask via `forms.md`, or fall back to a
labeled placeholder). Tier definitions carry **firmographics + psychographics + pain**, never
a dollar amount.

## Tier profile frame (fill one per ICP tier)

For each tier, build a deep profile using this generic frame. Pull the `name` and the
firmographic basics from your decided tiers; expand the rest from research.

```markdown
## [Tier name]

**Who**: [roles + firmographics: company size / stage / industry — no pricing]
**Where they hang out**: [channels and communities this tier actually uses]
**Decision trigger**: [the event/realization that makes them reach out]
**Industries**: [if relevant]

**What they say** (verbatim pains, in their own words):
- "[pain quote 1]"
- "[pain quote 2]"

**What resonates** (map each to one of your UCP pillars):
- [Pillar] → "[the proof framing that lands with this tier]"

**Content that converts this tier**:
- [format/topic that this tier engages with]
```

> Ground each profile in real evidence (sales calls, interviews, analytics) — not
> assumptions. Personas describe behavior and triggers, not demographics alone.

## Shared psychographic frame (adapt to your org)

Most orgs find their tiers share a common mindset. Capture the shared traits once in your
positioning notes, then reuse them in every tier profile and the qualification checklist. A
generic starting frame to adapt:

| Trait dimension | Question it answers |
|-----------------|---------------------|
| Expertise level | Aware of the problem but can't solve it alone? |
| Speed sensitivity | How much patience do they have for long timelines? |
| Outcome vs. tech focus | Do they buy the result or the technology? |
| Growth orientation | Are they building/scaling something? |
| Relationship expectation | Vendor transaction vs. partnership? |
| BS tolerance | Do they want proof over pitch? |

Replace these with your org's actual shared traits during the Positioning Intake.

## Messaging: same topic, multiple tier angles

The discipline: take one topic and write it once **per tier**, shifting only the angle,
vocabulary, and proof — never the underlying truth. Generic worked frame (fill from your tiers):

```markdown
### Topic: "[a single topic relevant to all tiers]"

**[Tier 1]**: [lead with this tier's pain → bridge to a key message → close with a pillar proof]
**[Tier 2]**: [same topic, this tier's vocabulary and pain, different proof emphasis]
**[Tier N]**: [same topic, this tier's register — avoid the off-key message you noted for it]
```

Rules:
- **Lead with pain**, in the tier's own words, before any claim.
- Use one of your key messages verbatim where it fits the tier's context.
- Close with a UCP pillar proof point — concrete, not a vague claim.
- Avoid the tier's "message that doesn't resonate" register (from your per-tier notes).

## UCP pillar reinforcement (prove, don't claim)

For each UCP pillar, build a small bank of proof-led examples across formats so the pillar
shows up consistently. Generic frame (one table per pillar):

```markdown
### [Pillar name] — meaning: [pillar.meaning]   proof: [pillar.proof]

| Format | Example (pain → proof, no vague claims) |
|--------|-----------------------------------------|
| Short post / tweet | [one-line proof framing] |
| Long post / story  | [a concrete before→after that demonstrates the pillar] |
| Reel / video       | [a to-camera or quick-cut demonstration of the pillar] |
| Carousel           | [a slide sequence that contrasts "without" vs "with"] |
```

Principle: a pillar earns belief through a **proof point**, not an adjective. "We do X
faster" is a claim; the pillar's proof point ("[concrete timeline/outcome]") makes it
credible. Always reach for the proof.

---

## Pillar-coverage principle

Across any campaign or quarter, **every** UCP pillar should get airtime, and **every** ICP
tier should get coverage. Use the Quarterly ICP Coverage Audit in [`forms.md`](forms.md) to
track both and surface gaps.
