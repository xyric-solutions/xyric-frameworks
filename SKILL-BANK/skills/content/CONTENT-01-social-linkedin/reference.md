# LinkedIn Content — Reference

Loaded on demand for detailed patterns, examples, and guidance. Brand voice, content
pillars, audience tiers, and key messages are an organization's decisions (see
[`ADAPT.md`](ADAPT.md)); the LinkedIn format facts and structural patterns below are
universal.

---

## The decisions this skill operates on

This skill needs the following from the organization (gather via the intake in
[`forms.md`](forms.md), record wherever your team keeps these decisions):

- **Brand voice** — tagline, 3–5 voice attributes, off-limits words/clichés.
- **Content** — 3–5 pillars (each a theme + core insight), active channels, and any
  founders (name, audience angle, personal voice) for founder posts.
- **Positioning** — brand promise, 5–9 key messages, audience tiers (name + description),
  and differentiators.

Resolve every concrete value from these decisions at use time. If something is not yet
decided, treat it as unanswered (ask via `forms.md`, or fall back to a labeled placeholder).

## Content Strategy Quick Reference

### Content Mix (60/30/10)

The 60/30/10 split is a widely-used content-balance convention, not org-specific. Fill the
"What It Includes" column from the organization's content pillars and voice attributes.

| Bucket | % | What It Includes | LinkedIn Examples |
|--------|---|------------------|-------------------|
| **Core** | 60% | Pillar-aligned — educational, thought leadership, case studies — with hooks/stories | Insight posts, methodology, pattern stories, industry commentary |
| **Personality** | 30% | Human side — BTS, hot takes, vulnerable moments, team dynamic | Team series, culture wildcards, behind-the-scenes, confession posts |
| **Wildcard** | 10% | Off-topic — trending topics, recommendations, polls | Analogies, weekend vibes, unrelated life lessons |

### Key Messages / Differentiators to Reinforce

Weave at least one of the organization's key messages (or differentiators) into every post.
Keep each message phrased the way the organization states it — don't re-invent it.

### Human Check (mandatory before every post)

| # | Check | If Fail |
|---|-------|---------|
| 1 | Read it out loud — sounds like a real person? | Rewrite casual, shorter sentences |
| 2 | At least ONE specific detail (name, number, place, year)? | Add a real reference |
| 3 | Would you send this as a message to a smart friend? | Cut formality |
| 4 | At least one imperfect, casual, or raw moment? | Add "Honestly," or an incomplete thought |
| 5 | First line stops the scroll? | Rewrite hook — punchier, specific |
| 6 | FREE of Dead Words (incl. the organization's off-limits list)? | Kill them all |

### Dead Words (never use)

| Dead Word | Replace With |
|-----------|-------------|
| "Leverage" | "Use" or remove |
| "Unlock" | "Find" / "Build" / "Get" |
| "Game-changer" | Say what actually changed |
| "In today's fast-paced world" | Delete. Start with the point. |
| "Let's dive in" | Delete. Just start. |
| "Here's the thing" | Delete or say it directly |
| "Synergy" / "Holistic" / "Paradigm" | Normal words |
| "I'm thrilled to announce" | "We just did [thing]" |
| "Excited to share" | Just share it |

Merge with anything on the organization's off-limits list.

### Design Brief Format

Every post needing a visual gets this. Use the organization's brand palette, typography, and
logo rules — never hardcode them here.

```
DESIGN BRIEF: [Visual type] | [Key text/quote] | [Mood/style]
Colours: brand palette    Fonts: brand typography    Logo: brand logo (bottom-right, subtle)
```

| Post Type | Brief Example |
|-----------|---------------|
| Contrarian Take | Quote card — key line as hero text, dark bg, light bold text |
| Myth Buster | Numbered list graphic — 3 items, each myth as a block |
| Story Hook | No graphic needed — text-only performs better |
| Team Series | Photo + welcome text overlay, brand frame |
| Carousel | X slides, slide 1 hook, last slide CTA/logo |
| Culture / BTS | Casual photo or phone shot — authenticity > polish |

## Thought-Leadership Pillars (how to expand each one)

Use the organization's pillar set. For each pillar the organization defines, expand it into
post material using this template:

```markdown
### Pillar: [pillar name]

**Core Insight**: [pillar's core insight]

**Content Angles**:
- [3-4 angles that explore the insight from different viewpoints]

**Example Hook**:
> [One scroll-stopping line that states the insight as a tension or surprise]
```

### Hook construction (universal)

A strong pillar hook usually does one of these:
- States a counterintuitive truth ("The tool wasn't the problem.")
- Names a gap between desire and reality ("Everyone wants X. Few do Y.")
- Opens a specific scene ("A client asked for [N] metrics. We asked one question.")
- Challenges a popular belief ("Most [thing] projects fail. Not for the reason you think.")

Build the specific numbers and references from real (anonymized) engagements — never invent
client outcomes.

## Anonymized Case Study Approach

### Why Anonymized
- Protects client confidentiality
- Allows flexibility in storytelling
- Creates premium mystique ("we work with clients who...")
- Enables composite stories from multiple engagements

### Pattern Language

| Instead Of | Use |
|------------|-----|
| "Our client [Named Co]..." | "A [industry] firm we worked with..." |
| "We built this dashboard for..." | "We recently encountered this pattern..." |
| "[Name], the CEO, told us..." | "A conversation with a client's leadership team revealed..." |

### Case Study Structure

```markdown
**The Pattern**: [Industry-agnostic observation]

**What We Saw**: [Specific situation without identifying details]

**What We Did**: [Methodology or approach]

**What Changed**: [Outcome — specific numbers if possible]

**What This Teaches**: [Broader lesson for the reader]
```

### Case Study Example (structure illustration — replace specifics with real engagements)

```
A pattern we've seen in growing businesses:

Reports that took days to compile. Decisions delayed by a week.
Leadership spending more time waiting for data than using it.

The fix wasn't a fancy new tool. It was restructuring how work flowed
from source to decision-maker.

Weeks later: same reports, automated. Decisions in hours, not weeks.

The lesson: Before buying new tools, audit your workflow.
Most bottlenecks are plumbing, not intelligence.
```

These illustrate post *structure* and tone. Replace the topic, pillar, and specifics with

### Example: Insight Post

```
[Hook line stating a tension the audience feels.]
[Second line that sharpens the tension.]

[The reframe: the real problem isn't what they think.]

[2-3 short paragraphs of substance — the pattern or method.]

[A concrete, anonymized observation with a real number.]

[The takeaway, stated as a principle.]

#PillarTag #DomainTag #TopicTag
```

### Example: Pattern Story (anonymized case study)

```
A pattern we see in [industry / business type]:

[The setup — what was happening, no identifying details.]
[Why it persists — everyone agrees, nobody has time.]

This is where most companies [do the common, failing thing].
Spoiler: [why it doesn't work].

What actually works:
[The approach — mapped to a method, not a tool.]

[The result — weeks of focused work, a specific before/after.]

The insight: [the broader lesson].

#PillarTag #DomainTag
```

### Example: Methodology Post

```
"[The common ask, in quotes]."

This is how most [topic] conversations start.
It's also why most [outputs] go unused.

[Reframe: the ask is an answer — what was the question?]

When we start an engagement, we ask: [the better question].

The [name]-first approach:
1. [Step]
2. [Step]
3. [Step]
4. [Step]

[Principle statement that closes the loop.]

#PillarTag #TopicTag #DomainTag
```

### Example: Industry Commentary

```
The uncomfortable truth about [trend]:

[The contrarian claim — most fail, and not for the obvious reason.]

[Evidence — what you've actually seen in practice.]

[A vivid, anonymized observation.]

The question isn't "[the hype question]?"
It's "[the readiness question]?"

[Sharp, memorable closing metaphor.]

#PillarTag #DomainTag #TopicTag
```

## Engagement Guidelines (universal)

### Responding to Comments
- Reply within the same business day.
- Add value, don't just thank.
- Ask follow-up questions when genuine.
- Never be defensive about pushback.
- Keep responses brief and confident.

### Proactive Engagement
- Comment thoughtfully on relevant industry posts.
- Engage with decision-makers' content (before selling).
- Add perspective, not self-promotion.
- Build relationships through value, not pitches.

## Post Timing

Optimal windows depend on the audience's time zones (infer from the organization's audience
tiers).

## Founder Content (LinkedIn)

When writing for a founder, use that founder's decided details (name, audience angle,
personal voice). Quick reference structure (fill from the organization's decisions — do not
assume a fixed set of founders):

| Dimension | Source |
|-----------|--------|
| Voice | the founder's personal voice |
| Pronoun | "I" (personal) |
| Audience | the founder's audience angle |
| Credibility | the founder's real, stated background |
| Reference the org | naturally — "We see this pattern...", "This is why we built it..." |

**Company page** always uses "we" — never "I" or founder names.

### Co-Founder Collab Formats (universal)

| Format | Description |
|--------|-------------|
| **Debate Post** | Two founders — same topic, opposite angles, published the same day |
| **Tag Team Thread** | One starts, the other adds — like a relay |
| **Joint Podcast Recap** | Shared clip or quote from a joint episode |
| **Split-Screen Reel** | Side-by-side reactions (also shareable cross-channel) |
