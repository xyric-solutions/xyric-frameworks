# Contributing Creative References

Creative references are useful only when they are searchable, attributable, and explain why they matter.

## Add A Reference

1. Add an entry to your `creatives-index.yaml`.
2. Include a source URL or a clear source note.
3. Add license or usage status.
4. Add attribution.
5. Add tags that describe style, medium, domain, and use case.
6. Write a short `why` note.
7. Store heavy files outside git and link the Drive file or folder if one exists.

## Required Fields

- `id`
- `title`
- `type`
- `source_url` or `source_note`
- `license`
- `attribution`
- `tags`
- `why`
- `usage_notes`

## Quality Bar

Good entries answer:

- What is this?
- Where did it come from?
- Can we reuse it, adapt it, or only learn from it?
- Why is it worth saving?
- What should an AI agent notice about it?

## Do Not Commit

- Raw licensed assets unless redistribution is allowed.
- Private client references.
- Screenshots containing sensitive data.
- Large binary files that should live in Drive.
